# PRD - ConsoleMVC (MVC 스켈레톤 PoC)

> **작성 기준**: 사용자 관점의 기능 요구사항. 기술 구현(클래스명, 라이브러리 등) 제외.

---

## 1. 제품 개요

반도체 시료 생산주문관리 시스템의 콘솔 기반 사용자 인터페이스 구조를 검증한다.  
주문 담당자와 생산 담당자가 콘솔 화면을 통해 시스템의 각 기능으로 진입하고 복귀할 수 있어야 한다.

---

## 2. 사용자 역할

| 역할 | 주요 업무 |
|------|-----------|
| 주문 담당자 | 고객 요청을 받아 주문 접수 |
| 생산 담당자 | 시료 등록, 주문 승인/거절, 생산 및 출고 처리 |

---

## 3. 기능 요구사항

### 3-1. 메인 메뉴

- 사용자는 시스템을 실행하면 메인 메뉴 화면을 볼 수 있어야 한다.
- 사용자는 메인 메뉴에서 번호를 입력하여 원하는 기능으로 진입할 수 있어야 한다.
- 사용자는 하위 메뉴에서 작업을 마친 후 메인 메뉴로 돌아올 수 있어야 한다.
- 사용자는 메인 메뉴에서 시스템을 종료할 수 있어야 한다.

### 3-2. 시료 관리

- 사용자는 시료 관리 메뉴에 진입할 수 있어야 한다.
- 사용자는 시료 관리 하위 메뉴(등록 / 조회 / 검색)를 선택할 수 있어야 한다.
- 사용자는 시료 관리 메뉴에서 메인 메뉴로 돌아올 수 있어야 한다.

### 3-3. 시료 주문

- 사용자는 시료 주문 메뉴에 진입할 수 있어야 한다.
- 사용자는 주문 접수에 필요한 정보를 입력하는 화면을 볼 수 있어야 한다.
- 사용자는 시료 주문 메뉴에서 메인 메뉴로 돌아올 수 있어야 한다.

### 3-4. 주문 승인/거절

- 사용자는 주문 승인/거절 메뉴에 진입할 수 있어야 한다.
- 사용자는 승인 또는 거절 처리를 선택하는 화면을 볼 수 있어야 한다.
- 사용자는 주문 승인/거절 메뉴에서 메인 메뉴로 돌아올 수 있어야 한다.

### 3-5. 모니터링

- 사용자는 모니터링 메뉴에 진입할 수 있어야 한다.
- 사용자는 주문 현황 및 재고 현황 요약을 볼 수 있어야 한다.
- 사용자는 모니터링 메뉴에서 메인 메뉴로 돌아올 수 있어야 한다.

### 3-6. 생산라인 조회

- 사용자는 생산라인 조회 메뉴에 진입할 수 있어야 한다.
- 사용자는 현재 생산 중인 시료와 대기 중인 시료 목록을 볼 수 있어야 한다.
- 사용자는 생산라인 조회 메뉴에서 메인 메뉴로 돌아올 수 있어야 한다.

### 3-7. 출고 처리

- 사용자는 출고 처리 메뉴에 진입할 수 있어야 한다.
- 사용자는 출고 가능한 주문 목록을 보고 출고 처리를 실행하는 화면을 볼 수 있어야 한다.
- 사용자는 출고 처리 메뉴에서 메인 메뉴로 돌아올 수 있어야 한다.

---

## 4. 비기능 요구사항

- 사용자는 잘못된 메뉴 번호를 입력했을 때 오류 메시지를 보고 다시 입력할 수 있어야 한다.
- 각 화면은 현재 위치(메뉴 이름)를 명확히 표시해야 한다.

---

## 5. 범위 외 (Out of Scope)

- 실제 데이터 처리 및 비즈니스 로직 (이 PoC는 화면 흐름 구조 검증이 목적)
- 데이터 영속성 (메모리 내 처리만)
- 실제 재고 계산, 생산 시간 계산
