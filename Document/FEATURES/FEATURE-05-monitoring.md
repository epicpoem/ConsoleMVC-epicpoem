# FEATURE-05: 모니터링

## 기능 목적

시스템 전체 현황을 한눈에 파악할 수 있도록 주문 상태별 건수와 시료별 재고 상태를 조회한다.

---

## 사용자 시나리오

1. 사용자가 모니터링 메뉴에 진입한다.
2. 상태별 주문 건수가 표시된다 (RESERVED / PRODUCING / CONFIRMED / RELEASE).
3. 시료별 현재 재고 수량과 재고 상태(여유 / 부족 / 고갈)가 표시된다.
4. 메인 메뉴로 돌아온다.

---

## 입력

- 없음 (진입 시 자동 조회)

## 출력

### 주문 현황
| 상태 | 건수 |
|------|------|
| RESERVED | N건 |
| PRODUCING | N건 |
| CONFIRMED | N건 |
| RELEASE | N건 |

- REJECTED는 표시하지 않음

### 재고 현황
각 시료별: 시료 ID, 시료명, 현재 재고, 재고 상태

| 상태 | 조건 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 재고 수량 0 |

- 재고 판단 기준: CONFIRMED + PRODUCING 상태 주문 총량과 비교

---

## 예외 처리 기준

- 등록된 시료 없을 시: 재고 현황 빈 목록 안내
- 주문 없을 시: 전체 건수 0으로 표시
