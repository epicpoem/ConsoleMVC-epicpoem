# FEATURE-03: 시료 주문

## 기능 목적

주문 담당자가 고객의 시료 요청을 받아 주문을 생성한다.  
생성된 주문은 RESERVED 상태로 접수되며, 승인/거절은 별도 메뉴에서 처리된다.

---

## 사용자 시나리오

1. 사용자가 시료 주문 메뉴에 진입한다.
2. 시료 ID, 고객명, 주문 수량을 입력한다.
3. 시스템이 주문번호를 자동 생성하고 RESERVED 상태로 주문을 등록한다.
4. 생성된 주문번호와 접수 확인 메시지가 출력된다.
5. 메인 메뉴로 돌아온다.

---

## 입력

| 항목 | 설명 |
|------|------|
| 시료 ID | 등록된 시료의 ID |
| 고객명 | 주문 고객 이름 (예: 삼성전자 파운드리) |
| 주문 수량 | 주문할 수량 (ea, 양의 정수) |

## 출력

- 자동 생성된 주문번호 (형식: ORD-{날짜}-{일별 4자리 순번}, 예: ORD-20260612-0001)
- 주문 접수 완료 메시지 (RESERVED 상태)

---

## 예외 처리 기준

- 존재하지 않는 시료 ID 입력 시: 오류 메시지 출력 후 재입력 유도
- 수량이 0 이하인 경우: 오류 메시지 출력 후 재입력 유도
