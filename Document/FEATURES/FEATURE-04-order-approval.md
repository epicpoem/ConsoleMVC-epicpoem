# FEATURE-04: 주문 승인/거절

## 기능 목적

생산 담당자가 RESERVED 상태의 주문을 검토하고 승인 또는 거절 처리한다.  
승인 시 재고 상황에 따라 주문 상태가 자동으로 분기된다.

---

## 사용자 시나리오

### 시나리오 A: 주문 승인 (재고 충분)
1. 사용자가 주문 승인/거절 메뉴에 진입한다.
2. RESERVED 상태의 주문 목록이 표시된다.
3. 사용자가 처리할 주문을 선택하고 승인을 선택한다.
4. 재고가 충분하면 주문이 CONFIRMED 상태로 전환된다.
5. 처리 결과 메시지가 출력된다.

### 시나리오 B: 주문 승인 (재고 부족)
1. ~ 3. 동일
4. 재고가 부족하면 부족분과 예상 생산량/생산시간 정보를 포함한 확인 메시지가 표시된다.
5. 사용자가 확인하면 생산라인에 등록되고 주문이 PRODUCING 상태로 전환된다.

### 시나리오 C: 주문 거절
1. 사용자가 주문 승인/거절 메뉴에 진입한다.
2. RESERVED 상태의 주문 목록이 표시된다.
3. 사용자가 처리할 주문을 선택하고 거절을 선택한다.
4. 주문이 REJECTED 상태로 전환된다.
5. 처리 결과 메시지가 출력된다.

---

## 입력

- 처리할 주문 선택
- 승인 또는 거절 선택
- 재고 부족 시 생산 진행 여부 확인

## 출력

- RESERVED 주문 목록 (주문번호, 고객명, 시료명, 주문 수량)
- 재고 부족 시 안내: 부족분, 예상 생산량, 예상 생산시간
- 처리 결과 메시지 (전환된 상태 포함)

---

## 예외 처리 기준

- RESERVED 주문이 없을 때: 빈 목록 안내 메시지 출력
- 유효하지 않은 주문 선택 시: 오류 메시지 출력 후 재입력 유도
